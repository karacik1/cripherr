Проектирование Model (Модель)
Создать класс, который будет содержать только логику шифрования.
Не использовать tkinter или любые другие библиотеки для GUI.
Использовать константы с понятными именами.
Написать методы, которые выполняют одну конкретную задачу.
Обработать возможные ошибки (например, некорректный ввод).


Проектирование View (Представление)
Создать класс, который будет содержать только элементы интерфейса.
Не содержать логику шифрования.
Использовать понятные имена переменных.
Сгруппировать элементы интерфейса логически.
Предусмотреть возможность изменения интерфейса (например, изменение темы, шрифтов).


Проектирование Controller (Контроллер)
Создать класс, который будет связывать Model и View.
Обрабатывать события (нажатия кнопок, ввод текста и т.д.).
Не содержать логику шифрования и элементы интерфейса.
Интеграция в главное приложение
Создать экземпляр контроллера при выборе шифра из списка.
Обеспечить правильное уничтожение предыдущего шифра.


Тестирование
Протестировать Model отдельно от GUI.
Проверить обработку ошибок.
----------------------------------------------------------------------------------------
Функция импортирует tkinter/tksheet? → Вероятно View
Функция обновляет данные в виджете? → View
Функция показывает сообщение об ошибке? → View (отображение)
Функция меняет цвет/размер/видимость виджета? → Точно View

Функция проверяет корректность данных? → Model (валидация)
Функция возвращает строку/число/список? → Вероятно Model
Функция вычисляет что-то математическое? → Model

Функция связывает событие с действием? → Controller
Функция вызывается по нажатию кнопки? → Вероятно Controller
Функция получает данные из полей ввода? → Controller (сбор данных)
